# 技术栈 Profile：frontend-vue3

## 适用范围

用于 Vue 3.0 前端需求规格化。

适用于本项目中与以下内容相关的需求：

- `web/` 前端页面。
- Vue 组件、路由、状态、表单、列表、表格和交互。
- 前后端接口对接。

## 规格化时必须关注

### 页面与路由

- 是否需要新增页面、调整已有页面或新增路由。
- 页面入口、跳转路径、返回路径是否明确。
- 页面是否需要空状态、加载状态、错误状态和无权限状态。

### 组件与交互

- 表单字段、默认值、必填项、校验规则是否明确。
- 按钮、菜单、弹窗、提示、二次确认是否明确。
- 列表、表格、分页、筛选、排序、刷新是否明确。
- 是否需要防重复提交、禁用态、加载态和错误重试。

### API 对接

- 前端调用哪些后端接口能力。
- 请求参数和响应字段是否明确。
- 接口失败、超时、无数据和权限不足时页面如何表现。
- 是否需要兼容接口暂未完成或字段缺失的情况。

### 用户体验与兼容性

- 是否需要移动端适配。
- 是否有浏览器兼容要求。
- 是否需要保留用户输入、筛选条件或页面状态。
- 页面文案是否需要统一口径。

## requirement-spec.md 中建议补充

- 用户角色中说明前端使用者。
- 业务流程中体现页面操作路径。
- 异常场景中列出接口失败、校验失败、空数据、重复提交。
- 验收标准中描述页面可观察结果。
- 兼容性要求中明确浏览器、分辨率或移动端要求。

## 验收标准提示

- 必填项缺失时阻止提交并提示字段。
- 接口失败时显示失败提示并保留用户输入。
- 查询无数据时显示空状态。
- 加载中禁用重复提交。
- 无权限时显示明确提示或隐藏不可用操作。

## 检查命令提示

规格化阶段不执行代码修改。后续如果进入开发并修改前端代码，至少执行：

```powershell
cd web
npm run build
```
