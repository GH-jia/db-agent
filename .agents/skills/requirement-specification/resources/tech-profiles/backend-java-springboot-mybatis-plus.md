# 技术栈 Profile：backend-java-springboot-mybatis-plus

## 适用范围

用于 Java Spring Boot + MyBatis-Plus 后端需求规格化。

当目标项目或目标模块是 Java 后端服务时使用。

## 规格化时必须关注

### Controller 需求

- 是否需要新增或调整 REST 接口。
- 请求 DTO、响应 DTO、状态码和错误响应是否明确。
- Controller 是否只做参数接收、基础校验和响应包装。

### Service 需求

- 核心业务规则是否明确。
- 事务边界是否明确。
- 是否涉及幂等、并发、状态流转、重复提交或补偿处理。
- 异常处理和错误码是否明确。

### MyBatis-Plus 数据访问

- 是否需要新增或调整 Entity、Mapper、Service、Wrapper 查询。
- 是否需要自定义 SQL。
- 分页、排序、筛选和批量操作规则是否明确。
- 是否涉及逻辑删除、自动填充、乐观锁或租户字段。

### 数据库需求

- 表结构、字段、索引、唯一性约束和默认值是否明确。
- 是否需要数据迁移脚本。
- 是否涉及 PostgreSQL、MySQL 或其他数据库差异。
- 是否存在历史数据兼容或补数据要求。

### 配置与集成

- 是否涉及 Spring 配置、环境变量、外部服务、消息队列或定时任务。
- 是否需要鉴权、审计日志或操作日志。
- 不在日志、接口响应或文档中暴露敏感信息。

## requirement-spec.md 中建议补充

- 接口需求中明确 Controller 能力、输入输出、错误码。
- 数据需求中明确实体、字段、表关系和查询条件。
- 业务规则中明确事务、幂等、并发和状态变化。
- 异常场景中覆盖参数非法、无权限、数据不存在、重复提交、数据库异常。
- 兼容性要求中明确 Java、Spring Boot、MyBatis-Plus、数据库版本或历史接口兼容。

## 验收标准提示

- 正常请求返回标准响应结构。
- 参数非法时返回明确错误。
- 无权限时拒绝操作。
- 数据不存在时返回业务错误或空结果，按需求定义。
- 重复提交不产生重复业务数据。
- 事务失败时数据状态保持一致。

## 检查命令提示

规格化阶段不执行代码修改。后续如果进入开发，应根据目标项目实际构建工具选择 Maven 或 Gradle 检查命令。
