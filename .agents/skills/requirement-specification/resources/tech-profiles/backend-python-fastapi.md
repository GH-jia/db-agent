# 技术栈 Profile：backend-python-fastapi

## 适用范围

用于 Python + FastAPI 后端需求规格化。

适用于本项目中与以下内容相关的需求：

- `api/`：FastAPI 路由层。
- `service/`：业务逻辑层，包含 LLM 调用、数据库 agent、知识库、向量库逻辑。
- `dao/`：数据库连接、ORM 模型和底层数据访问。
- `config.yaml` 与 `service/config.py`：配置结构和读取逻辑。
- PostgreSQL 业务数据库。

## 规格化时必须关注

### API 需求

- 是否需要新增或调整接口能力。
- 请求参数、响应结构、错误响应是否明确。
- 是否需要分页、筛选、排序、导出或流式返回。
- 路由层只处理请求参数、响应结构和路由组织，不承载复杂业务逻辑。

### Service 需求

- 业务规则是否已明确。
- 是否涉及 LLM 调用、数据库 agent、知识库或向量库。
- 是否存在模型调用失败、超时、空响应、非法响应等异常场景。
- 是否需要日志、错误提示或审计记录。

### DAO 与数据库需求

- 是否需要读取或写入 PostgreSQL。
- 涉及哪些业务对象、表、字段或查询条件。
- 是否需要新增表、调整字段或新增索引。
- 如涉及表结构变化，规格中必须说明需要同步维护 `db-agent.sql`。
- 不新增外键约束；表之间的关联字段只做业务关联。

### 配置与敏感信息

- 是否需要新增配置项。
- 如果修改配置结构，规格中必须说明同步更新 `service/config.py` 及相关调用。
- 不在日志、接口响应、注释或文档中暴露密码、token、API Key、数据库连接串。

## requirement-spec.md 中建议补充

- 接口需求表中列出接口能力、调用方、输入、输出和错误场景。
- 数据需求表中列出数据对象、字段、来源、用途。
- 异常场景中覆盖参数非法、无权限、无数据、数据库异常、模型调用失败。
- 兼容性要求中说明 Python、FastAPI、PostgreSQL 或历史接口兼容要求。

## 验收标准提示

- 查询成功时返回结构明确。
- 无数据时返回空结果，不返回系统异常。
- 参数非法时返回可理解的错误信息。
- 无权限时拒绝访问。
- 敏感信息不进入日志和接口响应。

## 检查命令提示

规格化阶段不执行代码修改。后续如果进入开发并修改后端代码，至少执行：

```powershell
.\.venv\Scripts\python -m compileall api service dao main.py logging_config.py
```
